    Chapter 8: Transforms and Transitions

  • CSS3 allows developers to make static elements on the page move around with just CSS and not a single line of JavaScript.
  • Transforms don’t work on inline elements.
  • Scaling and translating will not mess with the document flow. Whatever is being altered will not affect other elements on the page.
  • Transforms are applied in the oder that they are provided.
  • Only use or default to all when using a transition-property if you want animate every property. If you animate every property, then future supported properties could be inadvertently animated.
  • 200ms is generally considered the optimum time for a transition: anything slower will make the website seem slow, drawing generally unwanted attention to what was supposed to be a subtle effect. Anything faster may be too subtle.
  • With transitions there are no fine-grained control over any intermediate states.
  • Just because you can, doesn’t mean you should.

    • Chapter 12: Canvas, SVG, Drag and Drop

  • Canvas originated from a framework that Apple originally developed.
  • You must set the width and the height of the canvas in order for it to have a custom size. You cannot use CSS to change the canvas size.
  • Text that is inside of the canvas will only be displayed if the user's browser doesn't support it.
  • Just like in real life, we have to saturate the brush with a color. In JavaScript we must do the same thing when using the canvas.
  • The canvas coordinate system is different from the one that we all learned in math class.
  • Squares and rectangles have built in draw functions but in order to do other shapes we have to specifically tell the JavaScript where to draw on the canvas.
  • The canvas is not visible to screen readers. It does not create a DOM node so it's not very accessible.
  • SVG's can become very complex, so there are image editing software that will make the creation process much easier.
  • Canvas and SVG allow you to draw custom images to the screen, but each has their advantages and disadvantages so you have to pick one for the specific project that you will be working on.
  • To make an image draggable, you must add the draggable attribute to the image tag and set it to true.
  • Elements on the page by default cannot have items dropped on them, so we have to use preventDefault in order to allow the element to accept a drop.
