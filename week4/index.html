    Chapter 8: Forms

  • A button with the type of reset can be used to reset all the form fields. No additional code is needed.
  • The placeholder functionality decreases significantly the amount of code needed to be written to have placeholder text in an input field.
  • There are multiple types of input fields.
  • Using the input type password, the text entered into the input field will be obfuscated.
  • Checkbox information can be saved in an array and looped over to show them in the document.
  • There is a basic API for form validation, but for more complex validation a custom one needs to be created.
  • Forms are the best way to enter data into the browser so that it can be sent to the backend with JavaScript.

    Chapter 12: Object Oriented Programming

  • Class names should start with an uppercase letter.
  • JavaScript uses a prototypal inheritance model.
  • JavaScript's default is to make methods public, but they can be made private.
  • Objects cannot have both a get() and set().
  • You can avoid creating classes all together and create objects from other objects.
  • The main concepts of Object Oriented Programming are encapsulation, polymorphism, and inheritance.
  • Prototypes are live and this means that when properties and methods are added to it they are auto added to existing instances.

    Chapter 15: Modern JavaScript

  • The code in a module should have a single purpose. And functions should be grouped together based on their functionality.
  • ES6 added support for module use in JavaScript.
  • All code in modules is always in strict mode without the need for 'use strict' and there is no way to opt out of this.
  • Using * you can import all values and functions from a file and you can give them a namespace.
  • Having more than one default export will result in a syntax error.
